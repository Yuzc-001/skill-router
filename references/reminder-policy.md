# Reminder Policy

Skill Router is quiet by default.

This file exists only to keep the routing layer from turning into commentary.

---

## Speak only when needed

Speak when:
- the user explicitly asks which skill to use
- a routing decision materially changes the next action
- discovery actually happened
- vetting raised a real concern

---

## Stay quiet otherwise

Do not:
- announce that Skill Router is active
- narrate the routing sequence
- add routing headers or footers
- explain obvious skill choices
- remind the user that skills exist when that reminder changes nothing

---

## Output style

When Skill Router does speak, it should be:
- brief
- concrete
- neutral
- directly useful

Preferred shape: one line if one line is enough.

---

## Bottom line

If the routing layer is making itself noticeable without improving the next decision, it is doing too much.
