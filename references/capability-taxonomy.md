# Capability Taxonomy

A legacy reference.

Skill Router v0.5 no longer depends on a broad capability taxonomy as part of its main path.
This file remains only as an optional fallback reference for unusual ambiguity.

---

## When to read this file

Read this file only if both of the following are true:
1. routing is genuinely needed
2. the task is unusually ambiguous and you cannot identify the dominant path from installed reality, local defaults, platform signals, or the user’s explicit goal

If those conditions are not both true, do not read this file.

---

## What it is not for

Do **not** use this file to:
- classify every task before acting
- expand routing commentary
- justify a decision that is already obvious
- replace installed-reality judgment with category theater

If the next action is already clear, this file is unnecessary.

---

## Minimal fallback categories

If you truly need a category hint, keep it coarse:
- document work
- code work
- data work
- research work
- image / video work
- messaging / publishing work
- deployment / system work
- planning / strategy work

The point is not precision.
The point is just enough orientation to unblock the next real action.

---

## Bottom line

In v0.5, taxonomy is not the center.
If you can route without this file, you should.
